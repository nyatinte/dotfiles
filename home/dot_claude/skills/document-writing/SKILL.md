# AI Slop除去スキル

## 概要

日本語ドキュメントからAI Slop（AIが生成しがちな不自然な表現パターン）を検出し、自然な日本語に修正させる。textlintの@textlint-ja/textlint-rule-preset-ai-writingを使用して、誇張表現・機械的強調・過剰装飾を特定し、修正を行う。

## ワークフロー

### 1. ドキュメントの読み込み

Readツールで対象のMarkdownファイルを読み込む。

### 2. textlintチェックの実行

チェックスクリプトを実行する。

```bash
bash $CLAUDE_SKILL_DIR/scripts/check_document.sh <ファイルパス>
```

このスクリプトは `assets/.textlintrc` の設定を使用し、以下のAI Slopパターンを検出する。

| ルール名                  | 検出対象                                                        |
| ------------------------- | --------------------------------------------------------------- |
| no-ai-list-formatting     | 機械的なリスト装飾（「**重要**: テキスト」、絵文字接頭辞）      |
| no-ai-hype-expressions    | 誇張表現（「革命的」「ゲームチェンジャー」）                    |
| no-ai-emphasis-patterns   | 機械的な強調パターン（見出し内の太字、絵文字+太字の組み合わせ） |
| no-ai-colon-continuation  | 英語風コロンの使用（ブロック前の「:」）                         |
| ai-tech-writing-guideline | 技術文書作成のベストプラクティス                                |

textlintが未インストールの場合、スクリプトがインストール手順を表示する。

### 3. 結果の分析

textlintの出力を確認し、以下を特定する。

- 違反（error）: 修正必須の問題
- 提案（info）: 改善推奨の事項

### 4. 曖昧な内容の確認

以下に該当する内容がある場合、AskUserQuestionで確認する。

- 推測・推論に基づく記述
- 曖昧な表現
- ユーザーから明示されていない情報
- 検証が必要な主張

確認する内容は以下の通り。

- 事実の正確性
- ユーザーの意図
- 曖昧な表現を置き換える具体的な詳細

原則として、推測や憶測は行わない。不明な点があれば確認する。

### 5. 修正の適用

textlintの結果とユーザーからの確認内容に基づき、以下を実行する。

- Editツールで違反を修正する
- AI Slopパターンを自然な表現に置き換える
- 推測に基づく内容を削除するか、確認済みの事実に置き換える
- すべての情報が事実であり、ユーザーが確認済みであることを保証する

### 6. 品質の確認

修正後、チェックスクリプトを再実行して問題が解消されたことを確認する。

```bash
bash $CLAUDE_SKILL_DIR/scripts/check_document.sh <ファイルパス>
```

すべてのチェックに合格するまで、手順4〜6を繰り返す。

## ガイドライン

### 避けるべき表現

| カテゴリ       | 例                                                       |
| -------------- | -------------------------------------------------------- |
| 誇張表現       | 「革命的な」「ゲームチェンジャー」「究極の」「画期的な」 |
| 機械的な強調   | 見出し内の太字、絵文字と太字の組み合わせ                 |
| 過剰装飾リスト | 「**重要**: テキスト」「✅ 項目」「🔥 項目」             |
| 断定的な主張   | 「完全に解決」「すべての課題」「確実に」                 |
| 英語風コロン   | コードブロック前の「実行します:」                        |

### 望ましい代替表現

- 自然な日本語表現を使用する
- 装飾ではなく、文章構成で強調する
- シンプルなリスト記号を使用する
- 具体的で検証可能な主張を行う
- 日本語として自然な接続表現を使用する

### 曖昧さへの対処

以下に該当する場合、AskUserQuestionで確認する。

| 曖昧な表現                           | 確認内容             |
| ------------------------------------ | -------------------- |
| 「高速な」などの曖昧な技術仕様       | 具体的な数値・指標   |
| 「業界標準」などの未検証の主張       | 根拠・出典           |
| 「これにより」などの文脈不足         | 「これ」が何を指すか |
| 「実装された」などの主体不明の受動態 | 誰が実装したか       |

## 前提条件

textlintとプリセットをグローバルインストールする。

```bash
npm install -g textlint @textlint-ja/textlint-rule-preset-ai-writing textlint-rule-preset-ja-spacing
```

## リソース

### scripts/check_document.sh

設定済みルールでtextlintを実行する。依存関係が不足している場合、インストール手順を表示する。

### assets/.textlintrc

以下のルールを有効化したtextlint設定ファイル。

- @textlint-ja/preset-ai-writing: AI Slop検出ルール
- preset-ja-spacing: 日本語スペーシングルール（半角英数字周りのスペース、コード・リンク周りのスペース）
